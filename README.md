# vibe-travel-finance
## Телеграм-бот «Путешествия и деньги»

Проект представляет собой Telegram-бота, который помогает путешественнику:
- **вести несколько поездок** с названием, маршрутом, статусом (запланирована, активная, завершена),
- **планировать бюджет** поездки в любой валюте,
- **фиксировать расходы** по категориям (вручную и через QR‑строку чека),
- **вести мультивалютный кошелёк** и конвертировать валюту между счетами,
- **получать PDF‑отчёты** по всем поездкам и по каждой поездке отдельно,
- **получать предупреждение**, когда остаток бюджета по активной поездке падает ниже 50%.

Основная логика работы с БД и курсами находится в модуле `currency_app.py`, а взаимодействие с Telegram — в `bot.py`. Данные хранятся в SQLite‑базе `travel_and_money.db`.

---

### Стек технологий

- **Python 3.9+**
- **pyTelegramBotAPI** — работа с Telegram Bot API
- **requests** — HTTP‑запросы к сервису курсов валют
- **sqlite3** — встроенная БД для хранения поездок, расходов и баланса
- **colorama** — цветной вывод логов в консоль
- **python-dotenv** — загрузка конфигурации из файла `.env`
- **reportlab** — генерация PDF‑отчётов

---

### Структура проекта

- `bot.py` — Telegram‑бот:
  - главное меню и inline‑кнопки,
  - выбор поездки из списка, смена статуса (запланирована/активна/завершена),
  - создание новой поездки (маршрут + название + бюджет),
  - добавление расходов по категориям (с выбором валюты через inline‑клавиатуру),
  - приём строки с QR‑данными чека (параметры `t=` и `s=`),
  - просмотр баланса и расходов **по активной поездке**,
  - мультишаговый конвертер валют с выбором валют из списка и вводом собственного курса,
  - генерация и отправка PDF‑отчётов.
- `currency_app.py` — бизнес‑логика и работа с БД:
  - инициализация БД (`trips`, `balance`, `expenses`, `movements`),
  - сохранение поездок с названием, статусом и бюджетом,
  - сохранение расходов с привязкой к поездке,
  - логирование всех движений средств в таблице `movements`,
  - работа с мультивалютным кошельком (таблица `balance`),
  - запрос курсов валют из внешнего API,
  - расчёт сводок по балансам и расходам (общих и по конкретной поездке),
  - формирование PDF‑отчётов:
    - общий отчёт по всем поездкам (балансы, расходы, последние движения по пользователю),
    - отчёт по выбранной поездке:
      - **остаток бюджета** считается как *бюджет − траты* (по `expenses` этой поездки) и **не меняется от конвертаций**,
      - **балансы по валютам поездки** считаются как *бюджет (в валюте бюджета) + движения по поездке* (по `movements` с `trip_id`), при этом `trip_budget` в `movements` игнорируется, чтобы не задваивать бюджет,
      - отображаются **все движения средств** по этой поездке (включая конвертации) и **все валюты**, даже если итоговый остаток по валюте стал 0.
- `travel_and_money.db` — SQLite‑база данных (создаётся автоматически при первом запуске).
- `requirements.txt` — список зависимостей Python.
- `.env` — конфигурация и секреты (токен бота и ключи к внешним API).

### Настройка окружения

1. **Клонируйте или скопируйте проект** в удобную директорию.

2. **Создайте и активируйте виртуальное окружение (рекомендуется)**:

   ```bash
   python -m venv venv
   venv\Scripts\activate  # Windows
   ```

3. **Установите зависимости**:

   ```bash
   pip install -r requirements.txt
   ```

4. **Создайте файл `.env` в корне проекта** и пропишите переменные окружения (пример):

   ```env
   BOT_TOKEN=ваш_telegram_bot_token
   DB_NAME=travel_and_money.db
   EXCHANGE_RATE_KEY=ваш_api_ключ_для_сервиса_курсов_валют_или_оставьте_пустым
   EXCHANGE_RATE_URL=https://api.exchangerate.host/convert
   ```

   - `BOT_TOKEN` — токен Telegram‑бота от `@BotFather`.
   - `DB_NAME` — имя файла базы данных SQLite (по умолчанию `travel_and_money.db`).
   - `EXCHANGE_RATE_KEY` — ключ для получения курсов через `exchangerate.host` (нужен для авто‑получения курса; если ключ не задан, бот не сможет подставлять официальный курс, но конвертацию с ручным курсом можно выполнять).
   - `EXCHANGE_RATE_URL` — базовый URL для сервиса курсов валют (по умолчанию используется `https://api.exchangerate.host/convert`).

---

### Запуск бота

Запустите модуль `bot.py`:

```bash
python bot.py
```

При первом запуске:
- автоматически создаются таблицы в базе данных (`trips`, `balance`, `expenses`, `movements`),
- в консоль выводится статус о готовности бота.

После этого перейдите в Telegram к вашему боту и отправьте команду `/start`.

### Основные сценарии использования

- **Создание поездки**
  1. Нажмите кнопку «🌍 Новое путешествие».
  2. Если поездки уже есть — выберите нужную из списка или нажмите «➕ Новая поездка».
  3. Для новой поездки:
     - введите маршрут в формате: `Россия - Турция`,
     - затем введите **название поездки** (например `Отпуск в Китае весна 2026`),
     - затем введите **бюджет поездки** в нужной валюте (например `10000` или `1500 USD`).
  4. Бот сохранит поездку в БД, определит валюту стран, покажет курс и эквиваленты бюджета в валюте отправления и назначения.
  5. В списке поездок можно:
     - выбрать поездку,
     - изменить её статус (активна, завершена, запланирована),
     - удалить поездку (**все данные по ней удаляются каскадно**: расходы, движения, пересчитываются балансы),
     - запросить PDF‑отчёт по конкретной поездке.

- **Добавление расхода вручную**
  1. Нажмите «🛍️ Записать расход».
  2. Выберите категорию (Еда, Транспорт, Развлечения и т.д.).
  3. Выберите валюту расхода:
     - `RUB`,
     - валюта активной поездки,
     - или пункт «Другая валюта» — откроется inline‑список распространённых валют (с пагинацией).
  4. Введите сумму в выбранной валюте — расход сохранится в БД, баланс кошелька обновится, а при необходимости может прийти предупреждение о снижении остатка бюджета.

- **Добавление расхода по QR‑строке чека**
  1. Отправьте боту текст, содержащий параметры `t=` и `s=` (типичная строка из QR‑кода кассового чека).
  2. Бот извлечёт число из `s=...`, посчитает сумму как `число / 100` (копейки), сохранит расход в категории «Шопинг» и валюте `RUB` (параметр `t=` при этом не используется).

- **Просмотр «баланса» (фактически — суммарных расходов по валютам)**
  1. Нажмите «💰 Мой баланс».
  2. Если активная поездка не выбрана, бот сообщит, что нужно выбрать поездку.
  3. Если активная поездка есть, бот покажет:
     - название и маршрут активной поездки,
     - бюджет (в валюте бюджета),
     - сколько уже потрачено и сколько осталось (в валюте бюджета),
     - расходы по категориям **только для этой поездки**.

- **Конвертация валюты**
  1. Нажмите «🔁 Конвертер/Обмен».
  2. Выберите, **из какой валюты** вы хотите конвертировать (из имеющихся балансов, валюты поездки и популярных мировых валют).
  3. Выберите, **в какую валюту** конвертировать.
  4. Бот запросит официальный курс (если доступен через `EXCHANGE_RATE_KEY`) и попросит ввести:
     - только сумму (`100`), или
     - сумму и свой курс (`100 по 90,5`).
  5. После ввода данные кошелька обновятся, а если остаток бюджета по активной поездке упадёт ниже 50%, в чат придёт предупреждение.

- **PDF‑отчёт**
  - Кнопка «📄 PDF отчёт» формирует общий отчёт по всем поездкам (балансы, расходы, последние движения пользователя).
  - В меню конкретной поездки есть пункт «PDF по поездке», который формирует отчёт **строго по выбранной поездке**:
    - показывает бюджет поездки, траты и **остаток по бюджету** (*бюджет − траты*),
    - выводит **балансы по валютам именно этой поездки** (бюджет + движения по поездке, с учётом конвертаций),
    - обязательно отображает **все движения средств** по поездке,
    - показывает валюты **даже при нулевом остатке**.
